# PRD — Tsumugu Dictionary System

**Two stored definitions per word — a concise English gloss and a leveled monolingual Traditional-Chinese definition — plus a per-word store of 3–6 simple example sentences. The learner picks a default (English | 簡明中文) and toggles instantly, fully offline, because both definitions are already in the entry. The monolingual Chinese dictionary is the hard, novel part: definitions written with a defining vocabulary at or below a target band, batch-generated by an agent, OpenCC-guarded, and *band-verified by measurement* — the band constraint is never trusted from the prompt (definitional correctness is gated separately, by review + the custom-override layer; §5.3 Stage 6, §2 check 3). Everything is pre-baked into files the app consumes; no paid LLM API sits in the core loop, and the public engine stays data-free.**

> **Scope boundary with the sibling PRDs.** This PRD owns the **two definition layers** (sourcing the English data; *generating + leveling* the simple-Chinese definitions in batch), the **example-sentence store + its generation**, **storage / packaging / licensing**, the **toggle + default**, and **how hover + the encoding page consume them**. It is the **single source of the shared TS types** (`Definitions` / `EnDefinition` / `MonoDefinition` / `ExampleSentence`) and of the **toggle setting key** (`AppSettings.dictDefault`). The [[PRD-Encoding-Layer]] owns the SRS memory page (character story, mnemonics, related/confusables, FSRS review) that *consumes* this dictionary; per the shared contract §D/§E it **imports these types and the `dictDefault` key by name and does not redefine them** (a divergence exists today and is flagged for correction in §5.5 / §8). The [[PRD-Voice-Notes]] owns the local OSS TTS pipeline (Qwen3-TTS 1.7B, Serena); we *reuse* it to render per-sentence audio. Citations to `packages/engine/...`, `apps/web/...`, `packs/private/...`, `scripts/gen/...` are to the actual codebase as of 2026-06-09.

## 0. Decision log (decisions + overrides, on the record)

- **✅ Two definitions per word, both STORED, one shown at a time.** (1) English: a concise `·`-joined synonym gloss plus an optional one-sentence English explanation, with CC-CEDICT/kaikki senses finally populated. (2) Leveled monolingual Traditional Chinese: a definition plus a familiar-context illustration, written with a **defining vocabulary** at or below a target band. The reader popup and the encoding page render **one** at a time; the user sets a default and toggles instantly, offline. This resolves contract item A and the mockup's `English | 簡明中文` segmented control.
- **✅ The "leveled" claim is a BAND VERIFIER, not a prompt — and it is a NEW instrument, distinct from the CI metric.** We generate → segment → **band-ceiling check** against a single canonical level ladder (decomposition-aware against the band-N allow-list) → repair loop (≤ 3 iterations) → guard the failure modes the band metric can't catch → OpenCC `cn→twp` → stamp the *measured achieved band* as metadata. The existing CI metric (`packages/engine/src/ci/scorer.ts`, `personal/sources/measure_ci.ts`) measures **known-word coverage against a learner's store** (`getStatus`/`isKnown`) and has **no notion of a band ceiling**; the band check is a **new** function (`checkDefLevel`, §5.3 Stage 4) that the two metrics share only a segmenter with. "Measure, never guess" holds for the **band constraint**; definitional **correctness** is gated by review + the custom-override layer, not by measurement (§2 check 3, §5.3 Stage 6). Extends `scripts/gen/lib/verify.ts`.
- **✅ Defining-vocabulary ladder = the TOCFL 華語八千詞 list** we already carry (`packs/private/zh-hant/data/tocfl.json`, 14,814 rows, 7 tiers `TOCFL-1..TOCFL-7`, NAER/SC-TOP). We ship only the *computed, transformed* level-index we check against, and we keep that index **private (vault/private packs), never the public engine** — the list has no open license (§7, §10 R-5). The monolingual output's *level metadata is TOCFL-derived* and is **not** relicensed CC0 (§7, §10 R-5, §12 Q1, Q10).
- **✅ One canonical band ladder for the verifier.** TOCFL labels words `TOCFL-1..7`; the engine's `rankToBand` fallback labels out-of-list words `A1..C2`/`unranked` (`zh-hant/index.ts:117-124`). These are **two different label spaces with no defined ordering** as shipped. We define **TOCFL-1..7 as the single ceiling space** and a **total mapping** of every defining token into it (out-of-TOCFL tokens placed by a frequency-rank → TOCFL-band rule; truly unrankable tokens treated explicitly — §5.1, §5.3 Stage 4, §12 Q11). All `level` examples use the `TOCFL-N` scale; we drop the loose "/ A2" shorthand.
- **✅ Style template = 教育部國語小字典** (Taiwan MoE elementary dictionary): the register the simple-Chinese prompt imitates — single central sense, plain Traditional, no circularity. We seed 2–3 *short exemplars we author or that are uncopyrightably brief*; we do **not** redistribute its text, and we treat the MoE source text as do-not-derive (CC BY-ND — see below).
- **✅ License quarantine expressed in packaging — THREE regimes, per-source verified, no blanket CC0 claim.** Monolingual ZH definitions are **authored from scratch** (LLM text grounded on the headword + a non-BY-SA/non-BY-ND meaning anchor + the TOCFL allow-list); we do **not** seed/paraphrase MoEDict (its underlying MoE text is **CC BY-ND Taiwan 3.0** — NoDerivatives — so paraphrasing it is forbidden, strictly worse than ShareAlike, §7, §10 R-2). The LLM-authored definition **text** ships under **our own license** (CC0 *or* CC-BY for the text — §12 Q1); its **level metadata is TOCFL-derived** and shipped as a transformed private index, not relicensed. CC-CEDICT/kaikki (**CC-BY-SA**) stay in a **separate, attributed, BY-SA asset** — aggregated alongside, never *adapted into* the monolingual side, and **never fed into the monolingual generation context** (§5.3 Stage 1 input firewall). Tatoeba (**CC-BY 2.0 FR**) examples form a **third regime** with sentence-granular attribution and never sit inside any CC0-labeled asset. The build asserts each input's **verified, per-source** license against an allow-list (not a blanket "CC0-only"; §6.2). This is the load-bearing legal decision (§7, §10 R-2, R-5).
- **✅ Examples: 3–6 per word, AI-generated, simple/common/usable.** A **shared base set** is band-floor-gated only (no vault dependency → shareable). An optional **per-learner overlay** recycles the learner's known words (i+1) and lives in the vault, never in the shared asset (§5.4, §2 check 7). Tatoeba-first where a clean in-level CC-BY/CC0 sentence exists; LLM-generated fallback to guarantee coverage. Each sentence carries an English translation, optional Zhuyin/reading, optional per-sentence Serena audio, `source` + measured `level`, a `shared` flag, and `highlightSpans` for the inline accent-underline.
- **✅ Versioned, additive-where-possible schema evolution — with one deliberate, named non-additive widening.** Today `DictEntry.gloss` is one string and `PrebakedEntry.examples` is `string[]`. We move to `tsumugu/prepared-content@2` carrying `definitions:{en,zh}` and structured `examples:[…]`. This requires **widening `PreparedContent.schema` from the single literal `"tsumugu/prepared-content@1"` to the union `"tsumugu/prepared-content@1" | "tsumugu/prepared-content@2"`** (a non-additive change to an existing field — the word-store already has this union at `types.ts:261`; prepared-content does **not** yet) plus a new `PREPARED_CONTENT_SCHEMA_V2` constant. **`@1` files load unchanged** through a single normalizing reader — which does **not exist today**: the current loader does strict equality and a test explicitly asserts `@2` is *rejected* (`content/prepared.test.ts:62-64`); D0 must build that reader and rewrite that test (§6.2). Widening `examples: string[] → ExampleSentence[]` is breaking for any consumer reading raw `string[]`; D0 audits every consumer (§6.2, §10 R-8).
- **✅ Precedence unchanged in spirit, extended per field — but custom-example overrides require relocating `examples`.** `custom > prebaked > dict` (DECISIONS.md item 3, `hover.ts`). We extend `mergeHover` so each *new* field (each definition layer, structured examples) merges per-field with the same precedence and `sources[]` provenance. **Today `examples` lives only on `PrebakedEntry` (`types.ts:285`), not `DictEntry`; `custom?: Partial<DictEntry>` therefore cannot carry examples, and `mergeHover` hardcodes `const examples = prebaked?.examples` (`hover.ts:67`).** Enabling custom example overrides (use-case 4, Goal 2) requires **adding `examples?: ExampleSentence[]` to `DictEntry`** and rewriting that line to a `pick(custom?.examples, prebaked?.examples)` merge — a named D0 task (§5.7, §6.2). Until then examples resolve `prebaked` only.
- **✅ Default + toggle is one persisted preference, read by BOTH surfaces.** We replace the coarse `AppSettings.explanationLang` (`apps/web/src/state.ts:35`) with **`AppSettings.dictDefault: "en" | "zh"`**. Per contract §D this PRD **owns the key name**; the [[PRD-Encoding-Layer]] currently names it `defaultDefinition` (its lines 163/302/343) and **must rename to `dictDefault`** (§5.5, §8). The encoding-page toggle writes it; the reader popup reads it; the mockup's "default" pin is its visual reflection. The flip is a render switch, not a fetch (both definitions are already loaded).
- **✅ Offline packaging = lazy, level-sharded, out-of-bundle, with a stated size budget.** The dictionaries ship as **separate physical assets by license regime** (the from-scratch monolingual asset; the BY-SA bilingual asset; the CC-BY Tatoeba attributed example set if shipped), read at runtime from the granted vault folder via `VaultIO.readText` — never bundled into the engine. v1 keeps the existing JSON-shard path; at scale we adopt **sql.js-httpvfs SQLite** range-fetched from GitHub Pages. A small "core" band (TOCFL-1..2) may load eagerly under a stated MB budget (§6.3). `navigator.storage.persist()` is **best-effort** (eviction still possible).
- **✅ Hard rules inherited (non-negotiable).** Batch generation only (an agent writes files; the app consumes them); no paid LLM API in the core; engine stays DOM-free, network-free, **data-free**; zh-Hant output passes OpenCC (`cn→twp`); vault writes on explicit confirm; **no CC-BY-SA / CC-BY-ND data enters the Apache-2.0 public engine repo** (consistent with `/packs/private/` gitignored).

## 1. Problem

Tsumugu's reader and encoding page need to *explain a word*, and today's data model can explain it only one way. `DictEntry` carries a **single** `gloss` string whose own comment already admits it is overloaded — *"Primary gloss (monolingual-leveled default, **or** L2 toggle)"* (`packages/engine/src/types.ts:124`). The two readings are mutually exclusive in storage: no field holds **both** an English gloss and a leveled Chinese definition. `Sense` exists in the type (`types.ts:115`) but is never populated — every pack collapses CC-CEDICT / kaikki glosses into one `;`-joined string (`packs/private/zh-hant/index.ts:241`, `packs/private/vi/index.ts:154`). `PrebakedEntry.examples` is bare `string[]` (`types.ts:285`) — no per-sentence translation, reading, or audio, no target count, no known-word recycling. The settings toggle `explanationLang: "target" | "en"` (`apps/web/src/state.ts:35`) has nothing to flip between, because only one definition is stored.

Three concrete gaps follow (the contract calls these A/B/C):

1. **No second monolingual definition exists (gap A).** The learner cannot read a Chinese-only explanation of a Chinese word — the immersion technique that breaks the translation crutch. `explanation` is a freeform leveled blurb, not a *guaranteed-band-checked monolingual definition* enforced by a defining-vocabulary discipline.
2. **No real example store exists (gap B).** Examples are bare target strings scattered per prepared-content file, with no translation, audio, count discipline, or known-word recycling.
3. **The toggle is a label, not a switch (gap C).** The mockup (`mockup-3-encoding.html`) shows two definition cards side by side and a working `English | 簡明中文` pill with a "default" pin ("switch your default anytime") — and there is no stored pair to back it.

The monolingual Chinese dictionary is also the **novel** part of the product. No existing Chinese dictionary is graded to a learner proficiency scale: 教育部國語小字典 is plain but not leveled; 萌典/MoEDict is a redistribution of MoE data (and is **NoDerivatives**, §7), not leveled; TOCFL gives a word list, not definitions. A Chinese monolingual dictionary where every headword's definition is *band-checked* to use only words at or below a target TOCFL band does not exist off the shelf. We build it, and we make "leveled" a claim we can defend by measuring the band constraint.

## 2. Goals & success criteria (concrete, checkable)

**Data model — two definitions + structured examples, back-compatible:**

1. A single per-word entry stores **both** `definitions.en` and `definitions.zh` plus structured `examples[]`. Schema is `tsumugu/prepared-content@2`; **every existing `@1` file still loads** through a NEW normalizing reader (the current loader rejects `@2` — `content/prepared.test.ts:62-64`; that test is rewritten in D0). *Check: an `@1` fixture and an `@2` fixture both round-trip; a unit test asserts the `@1` upgrade fills `definitions.en.gloss` from the legacy `gloss` and `examples: string[]` → `[{text}]`; a replaced test asserts `@2` is now ACCEPTED and `@1` is upgraded; a compile-level test confirms no `string[]`-typed consumer reads `examples` after the widening.*
2. `mergeHover` resolves both definition layers and structured examples with `custom > prebaked > dict` precedence, per field — which for examples requires first relocating `examples` onto `DictEntry` and replacing the hardcoded `examples = prebaked?.examples` (`hover.ts:67`). *Check: a test where `custom.definitions.zh.gloss` wins over prebaked; and (after the D0 relocation) `custom.examples` wins over `prebaked.examples`; `sources[]` lists contributing layers in precedence order.*

**The monolingual Chinese dictionary (the hard part):**

3. **Band conformance (machine-checkable, necessary-not-sufficient).** For a fixed defining-vocabulary floor band N, ≥ 95% of generated `definitions.zh` use **only** vocabulary at or below band N *as measured by the new band verifier `checkDefLevel`* (decomposition-aware against the band-N allow-list), with the residual ≤ 5% either repaired, escalated one band (and tagged `levelEscalated`), or queued for review. **Correctness (semantic) is a SEPARATE gate, not covered by this number** — see check 3b. *Check: `pnpm gen verify` reports `defLevelViolations` + `achievedLevel` per file; a CI gate fails a file above the residual threshold.*
   - **3b. Definitional correctness (human + proxy gate).** A held-out spot-review of **N entries per band** (sample size and reviewer recorded) checks wrong-central-sense, proper-noun leakage, and "would a learner at band N read this with zero unknown words"; max acceptable wrong-sense/proper-noun rate and sign-off are stated in D2's exit. An automatic **round-trip proxy** (an LLM, given only the generated `definitions.zh`, recovers the headword from a small candidate set) provides one *measured* correctness signal beyond spot-sampling. *The 95% band number is necessary, not sufficient; it does not certify correctness.*
4. Zero `definitions.zh` string survives that contains Simplified characters: the OpenCC `cn→twp` guard runs over every zh definition + illustration + example, recording rewrites. *Check: `verify.ts` `openccChanges[]` covers the new zh fields; a Simplified-seeded fixture is caught and `--fix` rewrites it.*
5. No `definitions.zh` is **circular** (contains its own headword) or empty after stripping the headword. *Check: verifier circularity + emptiness tests.*

**The example-sentence store:**

6. Each in-scope word has **3–6** example sentences; each carries `{text, translation, reading?, audio?, source, level, shared, highlightSpans}`; the headword appears in every sentence and is marked in `highlightSpans`; every sentence passes the same band-check as definitions. *Check: verifier reports `exampleCount`, `headwordMissing`, `exampleLevelViolations`.*
7. The **shared base** example set is band-floor-gated only and carries **no dependency on any learner's `isKnown` set** (so it is shippable). A **per-learner overlay** (vault-only) recycles the learner's known words (i+1); the measured known-word recycle ratio is reported **on the overlay only**, against the vault word-store `isKnown`. *Check: shared sentences carry `shared: true` and pass band-floor with no vault read; the verifier computes the recycle ratio only over the per-learner overlay; D3's exit distinguishes "shared 3–6 generated" from "recycle-optimized for this vault."*

**Toggle + default + consumption:**

8. `AppSettings.dictDefault: "en" | "zh"` is one persisted preference, owned by THIS PRD (contract §D). The encoding-page toggle writes it; the reader popup and the encoding page both read it on load; flipping it moves the mockup's "default" pin and is instant + offline (no fetch). The [[PRD-Encoding-Layer]] consumes `dictDefault` (its `defaultDefinition` is renamed). *Check: switching on the encoding page and reopening the reader shows the chosen default; no network call fires; a grep confirms one settings key across both PRDs.*
9. **Hover renders a compact projection; the encoding page renders the full object — from the same entry.** Hover shows headword, reading, pos/level, the `dictDefault` definition (gloss + explanation/illustration), word audio, and the first 1–2 examples. The encoding page shows both definitions side by side + the full example list. *Check: one `ResolvedHover`/entry drives both; no second data source.*

**Offline packaging + open-core hygiene:**

10. Dictionary lookups are instant and fully offline once the pack data is present; the **public engine bundles zero dictionary data** (only public algorithms). *Check: `apps/web/src/packs/zhHant.ts` resolves via the runtime `BrowserDict`; with no vault dict present, lookups return `undefined` and the reader falls back to the pre-baked glossary (`apps/web/src/packs/index.ts:5-13`).*
11. Each shipped DATA asset carries its **own** license/attribution file (BY-SA for the bilingual asset; the chosen license for the from-scratch monolingual text; CC-BY attribution for any Tatoeba example set), and no asset is *adapted* from another. The build asserts the **verified per-source license** of every monolingual input against an allow-list (it **hard-fails** on a BY-SA/BY-ND source used as a *generation seed*; it *records* a non-redistributed reference), driven by a per-source provenance manifest. *Check: a `LICENSE`/`ATTRIBUTION` file sits beside each asset; the build's first test fixture is "a BY-ND MoEDict input as a seed → hard-fail"; Apache-2.0 governs only the engine code, and no CC-licensed data sits in the engine repo.*

## 3. Scope

- **vi monolingual dictionary.** vi keeps its English/bilingual gloss path (`packs/private/vi/index.ts`); a leveled monolingual *Vietnamese* dictionary is deferred (no TOCFL-equivalent ladder wired; frequency-only leveling today). - Any **paid LLM API in the core loop**, any **live in-app generation**, bundling licensed data in the public engine.

## 4. Users & use cases

**Primary: Wedge** (Intermediate+ zh-Hant; the monolingual Chinese dictionary is built for *his* immersion practice). Also any learner running the public engine with their own private pack; pack authors.

1. **Hover while reading.** Hover an unknown word → instant popup with the *default* definition (English or 簡明中文), reading, level, word audio, first example. Offline, no call (`apps/web/src/reader/reader.ts`).
2. **Open the encoding page.** Click a due word in Review → `mockup-3-encoding.html`. Both definition cards side by side; flip the `English | 簡明中文` pill to change the default everywhere; read the full 例句 list with per-sentence audio.
3. **Read Chinese-only.** Set default to 簡明中文 and read definitions that use only words at/below the floor band — breaking the English crutch, staying in the target language.
4. **Author a correction.** Override a wrong/awkward definition **or add/replace a custom example** via the vault custom layer; `custom > prebaked > dict` makes it win, with provenance shown. *(Custom-example override depends on the D0 relocation of `examples` onto `DictEntry` — §5.7, §6.2; until then only the definition is custom-overridable.)*
5. **Agent batch run.** Wedge (or an agent) runs the generation scripts that fill `definitions.zh` + the example store for a content file's unknown words, then `verify` to prove band conformance / OpenCC / coverage before the app ever reads it.

## 5. Core design

### 5.1 The two definition layers (the core duality)

Both definitions are **always stored** on the entry; the toggle picks which one renders. The model is a per-language definition map so it generalizes across packs (zh-Hant fills `en` + `zh`; vi fills `en` now). **These TS interfaces are the single source of truth; the [[PRD-Encoding-Layer]] imports them by name (contract §D/§E) rather than restating them.**

```ts
// NEW. packages/engine/src/types.ts — owned by THIS PRD; additive, versioned.
export interface EnDefinition {
  /** "·"-joined synonym gloss, e.g. "lively · bustling · buzzing with activity". */
  gloss: string;
  /** One-sentence English definition (mockup Card A .dx). Optional — the gloss alone may suffice. */
  explanation?: string;
  /** Finally populated: CC-CEDICT "/"-delimited defs / kaikki numbered senses. */
  senses?: Sense[];
}

export interface MonoDefinition {
  /** Leveled monolingual definition, e.g. "（形容）人多、又吵又有活力，讓人覺得很開心的樣子。". */
  gloss: string;
  /** Familiar-context illustration, also band-gated, e.g. "像夜市、廟會、過年的街上那種氣氛。". */
  illustration?: string;
  /**
   * Defining-vocabulary ceiling this was generated + VERIFIED against, on the
   * CANONICAL ladder: a "TOCFL-N" string (N in 1..7). We do NOT mix in the
   * A1..C2 freq fallback labels here — those are mapped into TOCFL-N at
   * check time (§5.3 Stage 4). e.g. "TOCFL-2".
   */
  level: string;
  /** Measured max token band actually present (verifier output; ≤ ceiling on a clean pass), as "TOCFL-N". */
  achievedLevel?: string;
  /** True when generation could not stay at/below `level` after the repair cap and was escalated. */
  levelEscalated?: boolean;
  monolingual: true;
  /**
   * Provenance of how the text was authored. "generated" = LLM-authored from
   * scratch (the only shipped path — we do NOT seed/paraphrase MoEDict, §7/R-2).
   */
  source?: "generated" | (string & {});
}

export interface Definitions {
  en?: EnDefinition;
  /** Target-language monolingual, leveled. Key is the pack's monolingual lang id. */
  zh?: MonoDefinition;            // zh-Hant pack; a vi pack would use its own key later
}
```

**Canonical level ladder (resolves the two-label-space gap).** `tocfl.json` labels in-list words `TOCFL-1..7` (445 / 448 / 480 / 1481 / 2709 / 4231 / 5020 rows respectively); the engine's `rankToBand` (`zh-hant/index.ts:117-124`) labels *out-of-list* words `A1..C2`/`unranked` from frequency, an **unordered-relative-to-TOCFL** space. The verifier uses **`TOCFL-1..7` as the only ceiling space** and maps every defining token into it: in-TOCFL tokens use their `TOCFL-N`; out-of-TOCFL tokens are placed by a documented **frequency-rank → TOCFL-band** rule (so the common-but-untoceled defining words — see §5.3 Stage 1 — land at a low band); truly unrankable tokens (no TOCFL, no freq) are treated as a **violation unless they decompose** into in-band sub-words (§5.3 Stage 4). `MonoDefinition.level`/`achievedLevel` are always `TOCFL-N` strings. (The mockup's "uses only A2 words" label renders as "TOCFL-2"; CEFR is display-only.)

**Mockup mapping** (`mockup-3-encoding.html`): Card A `.defcard.en` ← `definitions.en` (`.dg` = `gloss`, `.dx` = `explanation`); Card B `.defcard.zh` ← `definitions.zh` (`.dg.zh` Kaiti = `gloss`, the band label = `level` rendered as "TOCFL-N", `.dx.zh` = `illustration`). The "default" pin = `dictDefault`.

`DictEntry` and `PrebakedEntry` gain an optional `definitions?: Definitions`. `DictEntry` **also gains `examples?: ExampleSentence[]`** (the relocation that makes custom-example overrides possible — §5.7, §6.2). The legacy `gloss` string is **retained** for back-compat (it equals `definitions.en.gloss` after upgrade) so `@1` consumers keep working.

### 5.2 Sourcing the English dictionary

The English layer is *sourced*, not generated — it already largely exists. **It is a BY-SA-zone artifact** and stays out of the from-scratch monolingual asset.

- **CC-CEDICT** (`packs/private/zh-hant/data/cedict.json`, 122,410 entries, keyed Traditional) supplies `definitions.en.gloss` (the `·`-joined synonym list) and **`senses[]`** — the one substantive code change on this side is to **stop collapsing senses** into a single `;`-joined string (`packs/private/zh-hant/index.ts:241`) and instead populate the long-dormant `Sense` type from CC-CEDICT's `/`-delimited definitions. CC-CEDICT also supplies numbered pinyin (already used for reading).
- **kaikki.org / Wiktionary** (the vi pack already parses it via `packs/private/lib/kaikki.ts`) is an optional supplement for POS and additional English senses where CC-CEDICT is thin. Wiktionary is dual-licensed **CC-BY-SA + GFDL**; we elect **CC-BY-SA for data reuse** and meet its attribution/license-text requirements in the BY-SA asset's `LICENSE`/`ATTRIBUTION` (naming GFDL explicitly — §7). If meeting GFDL's terms proves awkward, kaikki is used only as a **non-redistributed reference** and dropped from the shipped asset.
- `definitions.en.explanation` (Card A's one-sentence definition) is short; it may be left empty (gloss alone is enough for the popup) or batch-generated. **Because it derives from the CC-CEDICT gloss, it is a BY-SA-zone artifact and is NOT placed in the monolingual asset** (§5.3 input firewall). It is **English output**, so it carries no OpenCC concern. (Whether we want it at all: §12 Q4.)

Both sources are **CC-BY-SA**; their data stays in the **BY-SA-quarantined** asset (§5.6, §6, §7) and is **never fed into the monolingual generation context** (§5.3 Stage 1).

### 5.3 Generating + leveling the simple-Chinese definitions (the novel core)

The monolingual Chinese definition is **authored from scratch, then band-proven**. Generation alone drifts above level; the band verifier is the source of truth for the *band constraint* (our standing rule: *measure the band, never guess it* — correctness is a separate human/proxy gate, Stage 6 + §2 check 3b). The pipeline is a loop, run by an agent in batch (Claude Code / Grok Build), extending the existing `scripts/gen/lib/skeleton.ts` (fill) + `scripts/gen/lib/verify.ts` (check). It is offline, deterministic, and cheap.

**Stage 0 — Choose the band (the single biggest design fork; surfaced in §12 Q1).**
- **(A) Fixed defining-vocabulary floor (default).** All definitions are written at a single low band (the **value is a §12 Q1 call**; the mockup says "A2", but at a low TOCFL floor the allow-list is small — TOCFL-1+2 = 893 words, through TOCFL-3 = 1,373 — versus Longman's ~2,000-word defining vocabulary curated *for definability*, so the floor likely needs to be **TOCFL-3 or TOCFL-4**, confirmed by the feasibility check in §11 D2). A *fixed* floor makes the **definitions** depend only on the headword + the floor (not on a learner's frontier), which is what lets the **definition text** be a shippable shared asset. Frequent per-entry escalation (Stage 5) would break that "fixed/shareable" premise, so the escalation/queue rate is a measured D2 exit gate.
- **(B) Per-learner frontier (alternative).** N tracks the learner's current band. This personalizes the definition but makes `definitions.zh` per-learner, not a shippable shared asset. **We default to (A)** and flag (B) as the fork.

**Stage 1 — Seed (deterministic) + the input firewall.** `buildSkeleton` already seeds each glossary slot with the pack's `gloss`, `reading`, `pos`, `level` (`skeleton.ts:50-65`). We extend it to seed an **empty** `definitions.zh.gloss: ""` with `level` set to the chosen floor band N, and to stage the **defining-vocabulary allow-list** for band N. **Input firewall (load-bearing for the license quarantine).** The zh-generation prompt context may contain **only**: the `headword`, its `reading`, `pos`, the band-N `allow-list`, and **a meaning anchor sourced from a non-BY-SA/non-BY-ND source** (e.g. a structured sense label we author, or a public-domain/our-own meaning hint). It **MUST NOT** be shown CC-CEDICT/kaikki gloss text, Wiktionary text, or **any MoEDict / MoE dictionary text** — MoE text is **CC BY-ND** (NoDerivatives, §7), so paraphrasing it is forbidden, and CEDICT/kaikki text is **CC-BY-SA**, so anchoring on it risks adapting-from a copyleft source (the exact R-2 contamination). The build-time provenance check (§6.2) enforces this firewall.

**Stage 2 — Generate (agent).** A new prompt `scripts/gen/prompts/dict-mono-zh.md` (sibling to the existing `content-prep.md`). Sketch:

> *Define **[headword]** (meaning anchor: **[non-BY-SA/non-BY-ND anchor]**) in Traditional Chinese (Taiwan, s2twp register) using **ONLY** words at or below TOCFL band **N**. Here is the allow-list of permitted words: [allow-list]. Constraints: one central sense only; ≤ **[X]** characters; **do not use the headword itself** anywhere (no circularity); use each defining word in its common, central sense (not a rare one to dodge the limit); plain everyday register. Then write **one** short familiar-context illustration under the same constraint. Match the register of these short exemplars: [2–3 brief, authored exemplars]. Output JSON: `{ "gloss": "...", "illustration": "..." }`.*

The agent fills `definitions.zh.gloss` + `illustration`. (No MoEDict text is in scope — §7, R-2.)

**Stage 3 — Segment (deterministic).** Tokenize each generated zh string. **Exactly one segmentation feeds the band lookup** (the lookup keys on exact headwords, so switching segmenters changes which tokens are levelable). **CKIP is NOT in the repo or deps** (`scripts/gen/` has no CKIP; §12 Q5); therefore the **D2 baseline is jieba seeded + HMM-off** (the GSM2 phantom-merge fix) feeding the band lookup. CKIP (Academia Sinica) is an **optional later upgrade** for Traditional adjudication; if adopted it becomes the *single* segmentation feeding the lookup (not a second stream beside jieba), with the env added to §8 and a D2 setup step. We drop the "CKIP adjudicates / jieba pre-filters" two-stream framing — there is one token stream into the band lookup. CKIP/jieba run in the agent/script environment, not the browser — generation-time tools, so the engine's DOM-free / data-free guarantee is untouched.

**Stage 4 — Band-check (deterministic, decomposition-aware) — a NEW instrument, `checkDefLevel`.** This is **not** a reuse of the CI metric (which scores known-word coverage against a learner store and has no band notion). New function `checkDefLevel`: (a) take the Stage-3 segmentation; (b) for each token resolve a band on the **canonical TOCFL ladder** (§5.1) — `TOCFL-N` from `tocfl.json` if present, else the documented freq-rank→TOCFL-band placement, else "unrankable"; (c) compare the band ordinal to the ceiling N; (d) apply **decomposition credit against the band-N allow-list** (not a learner store): a compound above N that decomposes into allow-list sub-words is **not** a violation, and an "unrankable" token that so decomposes is credited, otherwise it is a violation. Emit `defLevelViolations[]` and the measured `achievedLevel` (as `TOCFL-N`). Add both to `VerifyReport` and the `gen verify` CLI.

**Stage 5 — Repair loop (agent, ≤ 3 iterations).** Feed violations back:

> *These words in your definition exceed band N: **[violations]**. Rewrite the definition (and illustration) using only words at or below band N from the allow-list. Keep the same central sense; still no headword, still one sense, still ≤ [X] characters.*

If it still fails after 3 passes: **escalate** the ceiling by one band **for this entry only**, set `levelEscalated: true` and record the higher `achievedLevel`, and **queue for human spot-review** rather than silently shipping an unleveled definition. (Auto-escalate-and-tag vs hold-empty-for-review is a policy knob — §12 Q3. The aggregate escalation rate is a D2 feasibility gate — §11.)

**Stage 6 — Guard the failure modes the band metric can't catch (the documented Longman flaws).** These pass the band filter, so the verifier checks what it can and routes the rest to human + proxy review — **the strong "verified by measurement" claim holds for the BAND constraint, not for semantic correctness**:
- **Circularity** — the definition must not contain the headword (mechanically checkable).
- **Emptiness** — non-empty after stripping the headword (mechanically checkable).
- **Wrong-central-sense / proper-noun leakage** — an on-list word used in a rare sense, or a proper noun, still passes the band filter. These are **not** machine-certifiable; they are gated by the **§2 check-3b** spot-review (stated sample size, max acceptable rate, sign-off) **plus** the **round-trip correctness proxy** (an LLM, given only the generated `definitions.zh`, must recover the headword from a small candidate set) so correctness has at least one *measured* signal rather than spot-sampling alone. All wiring lives in `verify.ts`.

**Stage 7 — OpenCC guard (deterministic).** `verify.ts` already runs `scriptNormalizer` (OpenCC `cn→twp`) over every displayed string (`verify.ts:93-94`). We extend coverage to `definitions.zh.gloss`, `definitions.zh.illustration`, `definitions.en` (no-op for English), and every example field — recording rewrites in `openccChanges[]`, writing back on `--fix`. **The OpenCC guard and the `mergeHover` precedence must be extended field-by-field in lockstep with the schema** — these are the two places a forgotten new field would leak Simplified text or break provenance.

The achieved band is stamped into every entry as metadata, consistent with our "mark CI / source as metadata" discipline. The band verifier *is* the acceptance instrument for the band constraint of this dictionary; correctness is the human/proxy gate above.

### 5.4 The example-sentence store + its generation

**Shape.** `examples` becomes structured (target 3–6 per word):

```ts
export interface ExampleSentence {
  text: string;                 // target-language sentence (Traditional)
  translation: string;          // English
  reading?: string;             // optional Zhuyin/Pinyin
  audio?: string;               // optional per-sentence Serena TTS ref (PRD-Voice-Notes)
  source?: "tatoeba" | "generated" | (string & {});
  level?: string;               // measured band ("TOCFL-N", verifier)
  /**
   * True = part of the band-floor-gated SHARED base set (no vault dependency,
   * shippable). False/absent = per-learner overlay (vault-only, may recycle
   * the learner's known words; NOT in any shared/CC0-labeled asset).
   */
  shared?: boolean;
  /** Char spans to accent-underline; at minimum the headword occurrence(s). */
  highlightSpans?: { start: number; end: number }[];
}
```

This widens `examples?: string[]` to `examples?: ExampleSentence[]` under `@2` (the `@1` reader upgrades each bare string to `{text}` — §6.2). **Mockup mapping:** each `.srow` ← one `ExampleSentence` (`.scn` Kaiti = `text` with `highlightSpans` → `<em>`, `.sen` = `translation`, `.sau` 🔊 = `audio`). The field is **`highlightSpans: {start;end}[]`** (object-array) — the [[PRD-Encoding-Layer]]'s `focusSpans: [number,number][]` is the same concept under a different name/shape and is reconciled to this one (§8).

**Generation (batch).**
- **Shared base set (shareable).** Band-floor-gated, **no `isKnown` dependency**. Source order: **mine Tatoeba** (CC-BY 2.0 FR; *and* some CC0) for clean in-band sentences, then **LLM-generate** the remainder to guarantee 3–6. Filter Tatoeba by per-sentence license, run the **same band-check from §5.3**, require the headword in its target sense, dedupe, length-bound. **License routing (three regimes):** only **CC0** Tatoeba sentences may sit in a CC0-labeled asset; **CC-BY** Tatoeba sentences carry their own sentence-granular attribution (tatoeba.org link + the CC-BY-2.0-FR license statement + sentence author where required) and live in a **separate attributed (non-CC0) example asset**, never commingled under a CC0 label; **LLM-generated** sentences are **our output** (our own license), kept out of the BY-SA zone entirely. If sentence-granular CC-BY attribution proves not worth it, we **drop Tatoeba mining for the shared product** and rely on LLM-generated examples.
- **Per-learner overlay (vault-only, NOT shared).** LLM-generated sentences that prefer the learner's known words:

  > *Write **N** short, common, everyday Traditional-Chinese sentences using **[headword]** in the sense "[definitions.zh.gloss]". Prefer these words the learner already knows: [recycle list from the vault store]. Keep every word at/below band N. Give an English translation for each.*

  These read the vault `isKnown` set, so they are **personal data** and are tagged `shared: false`, live in the vault custom layer, and **never enter a shared/CC0 asset** (resolving the open-core-hygiene conflict in §2 check 7).
- **Tag + measure** each sentence's `source`, `level`, and `shared` flag. The **known-word recycle ratio** is computed against the vault store (`isKnown`) **only over the overlay**. Mark the headword span(s) in `highlightSpans`.

**Audio.** Per-sentence audio is rendered by the [[PRD-Voice-Notes]] pipeline (Qwen3-TTS 1.7B, Serena) into the vault, referenced by `audio`. Optional, additive, inert when absent — the reader falls back to Web Speech.

**Where it lives.** Examples stay *inline on the entry* in the prepared-content file in v1 (no separate global bank) — this keeps the `@1→@2` migration trivial and the consumption path unchanged; the per-learner overlay rides the vault custom layer. A shared word→sentences bank for cross-content reuse is an open question (§12 Q6).

### 5.6 Consumption: hover popup + encoding page (one source of truth)

One entry object feeds both surfaces; the popup renders a **compact projection** of the same object the encoding page renders in full.

### 5.7 Precedence (custom > prebaked > dict), restated for the new fields

DECISIONS.md item 3 and `hover.ts` already fix `custom > prebaked > dict`. The user's custom layer lives in two places, unchanged: per-word `WordEntry.custom?: Partial<DictEntry>` in the vault store (`types.ts:237`) and the optional `<lang>/data/custom.json` in each private pack (`zh-hant/index.ts:248-253`).

**Custom-example overrides need a relocation first (named D0 task).** Today `examples` lives **only on `PrebakedEntry` (`types.ts:285`), not `DictEntry`**, so `custom?: Partial<DictEntry>` *cannot* carry examples, and `mergeHover` hardcodes `const examples = prebaked?.examples` (`hover.ts:67`) — meaning use-case 4's "add a custom example" and Goal 2's "custom example wins" are **not possible as the code stands**. D0 therefore: (a) adds `examples?: ExampleSentence[]` to **`DictEntry`** (so `Partial<DictEntry>` can hold it); (b) rewrites `hover.ts:67` to `pick(custom?.examples, prebaked?.examples)`; (c) records this as a back-compat consideration (no `@1` file has `DictEntry.examples`, so the addition is additive on disk). With that done, `Partial<DictEntry>` carries `definitions` and structured `examples`, so a learner can override *just* the zh definition, *just* an example, or add a custom example — and it wins per field with provenance shown. **Until (a)–(b) land, custom overrides cover `definitions` only and examples remain prebaked-only** (the §12 Q6-adjacent scope-down option).

### 6.2 Schema evolution + back-compat (the load-bearing engineering)

`PreparedContent.schema` is today a **single literal** `"tsumugu/prepared-content@1"` (`types.ts:300`, constant `PREPARED_CONTENT_SCHEMA` at `types.ts:317`), and the loader `packages/engine/src/content/prepared.ts` does **strict equality** (`if (x.schema !== PREPARED_CONTENT_SCHEMA) return false`) so `parsePreparedContent` **throws on anything not exactly `@1`** — and there is a currently-passing test asserting `@2` is **rejected** (`content/prepared.test.ts:62-64`). The word-store, by contrast, already has real version tolerance: a union `"tsumugu/word-store@1" | "tsumugu/word-store@2"` (`types.ts:261`) plus `WORD_STORE_SCHEMA_V1` (`types.ts:270`). **Prepared-content does NOT have this yet; D0 must build it.** The "single normalizing reader" this PRD relies on is named but absent from the repo today.

Concrete D0 deltas (the load-bearing engineering, stated explicitly so the work is not under-scoped):

1. **Widen the schema literal to a union (non-additive change to an existing field).** `PreparedContent.schema: "tsumugu/prepared-content@1" | "tsumugu/prepared-content@2"`, mirroring `types.ts:261`; add `export const PREPARED_CONTENT_SCHEMA_V2 = "tsumugu/prepared-content@2" as const` (mirror `types.ts:268-270`). Keep `PREPARED_CONTENT_SCHEMA` as the `@1` accepted-on-load constant.
2. **Build the normalizing reader** (`packages/engine/src/content/schema.ts`, new) that accepts `@1` or `@2` and returns a canonical in-memory `@2`:
   - `@1` → fill `definitions.en.gloss` from legacy `gloss`; `definitions.en.explanation` from legacy `explanation`; `examples: string[]` → `examples: [{text}]`. No `definitions.zh` (none was generated) → the toggle simply has no zh card for old content until regenerated.
   - `@2` → pass through.
3. **Change the guard + parser to accept both versions.** Update `isPreparedContent` / `parsePreparedContent` in `content/prepared.ts` to accept `@1` **or** `@2` and route `@1` through the upgrade. **Rewrite `content/prepared.test.ts:62-64`** — its assertion that `@2` is rejected is the regression we are deliberately removing; replace it with an *accept-`@2`* + *upgrade-`@1`* test.
4. **Relocate `examples` for custom overrides** (§5.7): add `examples?: ExampleSentence[]` to `DictEntry`; rewrite `hover.ts:67` to `pick(custom?.examples, prebaked?.examples)`.
5. **Consumer audit for the `string[] → ExampleSentence[]` widening (breaking).** Enumerate every reader of `PrebakedEntry.examples` (engine, the Anki exporter `anki/exporter.ts`, `apps/web`) and confirm each goes through the normalizing reader *before* the type widens, so no consumer still sees raw `string[]`. Add a **compile-level test** that an `@1` file's widened `examples` are upgraded before any `string[]`-typed consumer touches them, plus the round-trip fixture test for both schemas.
6. **Writers stamp `@2`.** Generation scripts emit `@2`; the reader emits `@2` on any re-save. `@1` files on disk keep loading forever; they are upgraded lazily in memory, never rewritten without a confirm.
7. **Two checks extended field-by-field, in lockstep with the schema:** (a) the OpenCC guard in `verify.ts` must cover every new *displayed* zh string; (b) `mergeHover` precedence must cover every new field. These are the two places a forgotten field would leak Simplified text or break provenance.

**D0 exit check:** the prepared-content guard **accepts `@2`** and the old reject-`@2` test is replaced by an accept-`@2` + upgrade-`@1` test; `mergeHover` resolves custom examples after the relocation; no `string[]`-typed consumer reads `examples`.

**Build-time license assertion (replaces the unsound "CC0-only" blanket).** The build encodes the **verified license of each input** (a per-source provenance manifest with license tag + source URL + retrieval date), checked against an allow-list. It **hard-fails** when a CC-BY-SA or CC-BY-ND source is detected as a **generation seed** for the monolingual pipeline (and records — does not fail — a non-redistributed *reference*). **The first test fixture is "MoEDict's real status (CC BY-ND text) used as a seed → hard-fail,"** turning the previously-false "assert inputs are CC0-only" guarantee into a true, auditable one.

### 6.3 Offline packaging + lazy-load (with a stated size budget)

- **v1 (now): JSON shards out of bundle.** The dictionaries are JSON read at runtime from the vault via `VaultIO.readText`, lazily + cached (current pattern). Each entry is **band-tagged**, so we ship a small **core band** (TOCFL-1..2) eagerly and lazy-load higher bands on first miss. If the file is absent, every lookup yields `undefined` and the reader falls back to the pre-baked glossary.
- **Size budget (concrete, checkable).** Eager **core band (TOCFL-1..2, ~893 headwords)** target **≤ 2 MB ungzipped** (≈ ≤ 0.6 MB gzipped). Total per-pack ungzipped ceiling **≤ 40 MB** before we *must* be on SQLite. **SQLite migration trigger (Q8 candidate number): switch JSON → sql.js-httpvfs once a single shipped asset exceeds ~25 MB ungzipped OR ~50k entries** — whichever first. The actual numbers are measured in D5 and recorded; these are the budget, not a guess to ship blind.
- **At scale: sql.js-httpvfs SQLite.** CC-CEDICT (~122k) is tens of MB — borderline-annoying as eager JSON, trivial as range-fetched SQLite. The monolingual from-scratch set's size is **conditional on how many headwords we author** (it is *not* a "~160k MoEDict-derived" set — MoEDict is neither used nor in the repo; §8). We build read-only `.sqlite` assets **per license regime** (the from-scratch monolingual asset; the BY-SA bilingual asset; the CC-BY Tatoeba example asset if shipped), indexed on headword + reading + band, served from GitHub Pages (Range-capable). **Per-lookup transfer is measured in D5** (expected on the order of a few KB for a headword/reading/band index lookup, not assumed at exactly 1 KB).
- **True-offline layer (optional, best-effort).** For plane-mode, a one-time "download pack" pulls the `.sqlite` into OPFS / IndexedDB-backed VFS and calls `navigator.storage.persist()`. **`persist()` is best-effort: the browser may refuse it and may still evict** — so "true offline" carries that caveat; the JSON-shard fallback remains.
- **Physical assets = the license quarantine in packaging** (§5.6, §6.1, §7): each asset (from-scratch monolingual, BY-SA bilingual, CC-BY Tatoeba) carries its own license/attribution file alongside; the build asserts they are *aggregated*, not adapted into each other, and that no CC-licensed asset enters the engine repo. The TOCFL level-index ships **private only**.

## 7. Technical foundations (real datasets + licenses, researched 2026-06-09)

**Defining-vocabulary prior art.** Longman's LDOCE (1978) wrote every definition with a fixed ~2,000-word controlled Defining Vocabulary (derived from West's General Service List), curated *for definability*; a definition's words must be easier than the headword, in their central sense. Documented failure modes — imprecision, circularity, using a defining word in a non-central sense to dodge the constraint — are exactly what our verifier (band check) + human/proxy gate (§5.3 Stage 6) catch. **Caveat on the TOCFL floor:** the bottom TOCFL bands are an everyday-proficiency list, not a curated definitional primitive set, and are smaller than Longman's 2,000 (TOCFL-1+2 = 893; through TOCFL-3 = 1,373; through TOCFL-4 = 2,854), so the chosen floor and the allow-list composition need the §11 D2 feasibility check (§12 Q1, Q12). The Chinese **register** template is 教育部國語小字典 (Taiwan MoE elementary dictionary, `dict.mini.moe.edu.tw`); we imitate its register from **short authored exemplars**, never its text. The **leveled ladder** is TOCFL.

## 8. Constraints & dependencies
