# PRD — Tsumugu Dictionary App

**A dedicated, offline-first character dictionary inside Tsumugu — every entry at Expert Edition depth, structured on the Outlier Functional Component Framework, batch-generated by an agent, voiced section-by-section by Serena (Qwen3-TTS). One character per page: FORM → typed COMPONENTS → MEANING TREE → STROKE ORDER → Expert evolution (ancient forms, paleographic narrative, sound-series / system links) → WORDS / SENTS tabs. The learner gets predictive ability and long-term recall without folk-radical stories; the engine stays data-free; Outlier PDFs stay private reference only.**

> **Product boundary.** This is the **character dictionary app** — lookup by single hanzi (`#/dict/造`). [[PRD-Dictionary]] remains the **word dictionary** (multi-character headwords, English | 簡明中文 toggle, leveled defs, example sentences). [[PRD-Encoding-Layer]] is the **SRS memory page** for words you're actively drilling; it *imports* a compressed projection of this app's Expert entry for the character story. This PRD **supersedes** the essentials-only scope in [[PRD-Character-Structure]] — Expert depth is the default ship target, not an optional block.

## 0. Decision log

- **✅ Expert Edition depth by default.** We ship the full shape shown in `expert-details.jpg` (造): Essentials sections (FORM, COMPONENTS, MEANINGS, STROKE ORDER) **plus** the Expert expansion (evolution narrative, ancient-form comparisons, scholarly notes, disambiguation of apparent sound/meaning relations). No "Essentials tier" product split — one entry schema, one renderer.
- **✅ Pedagogy grounded in your Outlier Masterclass PDFs.** Generation prompts encode the course sequence: **Unit I** (Rules of Memory → Three Attributes → Surface vs Deep Structure) before **Unit II** (Form → Meaning → Sound → Sound Series → Empty/Corruption) before **Unit III** (Meaning types & trees → IME → Pipelining). Private PDFs in `personal/references/outlier/pdfs/` are **non-redistributed agent reference**; output is Tsumugu-authored structured JSON + prose in our voice, not paraphrased Outlier paragraphs.
- **✅ AI-generated, batch-only, verified.** Agent (Claude Code / Grok Build) fills `dict-entry@1` JSON via staged prompts; `gen verify dict` gates grounding, component typing, OpenCC, and schema completeness. No paid API in the app loop.
- **✅ Serena voice on every readable section.** Reuse [[PRD-Voice-Notes]] (Qwen3-TTS 1.7B, Serena): per-section MP3 refs on FORM, each COMPONENT blurb, MEANINGS list, Expert narrative blocks, example sentences. Web Speech fallback when absent.
- **✅ Dedicated app surface.** New route `apps/web` → `#/dict/<char>` (and search/browse UI). Not buried inside reader hover — reader/encoding *link here* ("Open in Dictionary").
- **✅ Per-character canon, NFC-keyed.** One JSON per character under `packs/private/zh-hant/data/dict-app/` (or vault mirror). Multi-char words assemble char entries for a summary strip; word-level defs stay in [[PRD-Dictionary]].
- **✅ Open-core hygiene.** Public engine: types, resolver, renderer components only. All entries, audio, ancient-form image refs, and PDFs stay gitignored private.

## 1. Problem

You own the Outlier reference material and the Tsumugu engine, but there is no **character dictionary product** that combines:

1. **Expert-depth structure** (the right column of `expert-details.jpg` — not the thin Essentials phone mockup alone).
2. **Functional components done right** (form / meaning / sound / empty + corruption), per the Masterclass progression in PDFs 01–08.
3. **Meaning trees** with original vs extended vs modern senses (PDF 09), not a flat gloss list.
4. **System links** — sound series, same-component neighbours (PDF 07).
5. **Voice** — hear FORM and Expert narrative in natural Taiwan Mandarin while reading.
6. **Offline batch generation** at scale, correctable, integrated with your word store / SRS / reader.

Today's Tsumugu has CC-CEDICT glosses, a draft word-dictionary PRD, and an encoding page that still risks folk decomposition (`鬥+市` for `鬧`). None of that is a Pleco/Outlier-grade **character page**.

## 2. Pedagogy map (from your PDFs → entry schema)

The generation pipeline **must follow this order** (agents are instructed explicitly; verify checks section presence).

### Unit I — First principles (PDFs 01–03)

| Lesson | PDF | Teaches | Entry field |
|--------|-----|---------|-------------|
| 1 | `01-RulesofMemoryTraditional` | Higbee rules; **6 levels of understanding** (strokes → components → functional components → how they function → corruption → original meaning → full story); goal = predictive ability + long-term recall | `pedagogy.levelOfUnderstanding` (1–6 target: ship at ≥4 for Expert) |
| 2 | `02-ThreeAttributesTraditional` | Writing = **form + sound + meaning**; speech vs writing analogy | `attributes: { form, sound, meaning }` summary line |
| 3 | `03-SurfaceStructureandDeepStructuretrad` | **Surface** (strokes, radicals, standards) vs **Deep** (functional components, sound/meaning logic); non-functional vs functional | `surfaceNote?` (stroke-order only); deep structure in COMPONENTS |

**Design rule from Lesson 1 chart** (`Screenshot 2023-12-14 at 12.10.21.png`): entries must never present stroke lists or arbitrary chunking as the primary story. Functional components are the minimum viable explanation; Expert adds corruption + original meaning + full evolution.

### Unit II — Component types (PDFs 04–08)

| Lesson | PDF | Teaches | Entry field |
|--------|-----|---------|-------------|
| 4 | `04FormComponentstrad` | **Form component** = depicts something; meaning via picture-like quality; form ≠ meaning (e.g. 大 as form of adult silhouette) | `components[].role: "form"` + `function` |
| 5 | `05MeaningComponentstrad` | **Meaning component** = contributes acquired sense, not depiction; semantic series (力 → effort/power) | `role: "meaning"` |
| 6 | `06SoundComponentstrad` | **Sound component**; rebus principle; OC/MC/Mandarin drift; disambiguation (背 north vs back) | `role: "sound"` + `soundHint` + `soundNote` |
| 7 | `07SoundSeriestrad` | **Sound series** — characters sharing a sound component; prediction across series | `systemLinks.soundSeries[]` |
| 8 | `08EmptyComponentstrad` | **Empty** = distinguishing marks + **corruption**; don't invent sound/meaning for corrupted pieces | `role: "empty"` + `isCorrupted` + `corruptionNote` |

**Glossary** (`Chinese-character-glossary.pdf`): generation may use terms (analogy, bamboo script, character meaning vs component meaning, Old/Middle Chinese) in `expert.glossaryRefs[]` — definitions inlined once in app help, not per entry.

### Unit III — Meaning & learning strategy (PDFs 09–12)

| Lesson | PDF | Teaches | Entry field |
|--------|-----|---------|-------------|
| 9 | `09AllAboutMeaningtrad` | Character meaning vs component meaning; original / basic modern / derived / sound-loan; **meaning tree** | `meaningTree` |
| 10 | `10TheIMEMethodtrad` | IME / input method context (optional UI tip) | — (onboarding only) |
| 12 | `12ThePipeliningStrategytrad` | Don't learn form+sound+meaning simultaneously; pipeline study modes | `studyModes` in app UI (§5.6) |

**Cursive lessons 1–8** (`Lesson*-Cursive.pdf`): optional `cursiveNote?` + link to stroke tab variant; deferred post-v1 unless you want them in STROKE tab.

### Expert Edition layout (`expert-details.jpg`, character 造)

| Section | Essentials (left) | Expert (right) — **we ship all** |
|---------|-------------------|----------------------------------|
| Header | 造, zào (4), zhuyin | same |
| Tabs | DICT · STROKE · CHARS · WORDS · SENTS | same + 🔊 per section |
| FORM | One paragraph: composition + original sense hint | same, plus |
| COMPONENTS | Numbered; role per component (form/sound); tap → component entry | Expanded per-component entries; disambiguation when apparent components mislead |
| MEANINGS | Numbered senses; (orig.) marked | **Meaning tree** with sense relations |

| Expert-only | "Learn more" teaser | Full **evolution** prose; **ancient forms** (images or Unicode refs); compare to related chars (e.g. 告); scholarly attribution; OC/MC notes |

## 3. Goals & success criteria

**App (client-side, offline):**

1. Open `#/dict/造` → render full Expert entry synchronously from local `dict-entry@1` JSON + optional section audio; **zero network** after vault grant. *Check: File System Access test double shows no fetch post-load.*
2. Tabs DICT / STROKE / CHARS / WORDS / SENTS switch without remounting the entry; DICT tab order matches `expert-details.jpg`: FORM → COMPONENTS → MEANINGS → Expert block. *Check: DOM order test against fixture.*
3. Each COMPONENT row shows **role badge** (form | meaning | sound | empty) and links to `#/dict/<glyph>` for that component character. *Check: 造 → 辶, 告 navigable.*
4. 🔊 on FORM, each component, MEANINGS, Expert narrative plays Serena MP3 when present; Web Speech fallback when absent. *Check: player tests mirror PRD-Voice-Notes.*
5. Search: paste character or pinyin → navigate; browse by radical **or** by sound series (system index). *Check: search 造, zao, sound-series index for 告 series.*

**Data & generation (batch):**

6. Every generated entry validates against `dict-entry@1` schema; `gen verify dict` fails on: missing component role, circular folk story without `grounding`, Simplified leakage, empty `formExplanation`, Expert block missing for characters with `pedagogy.levelOfUnderstanding` target ≥5. *Check: CI fixture suite.*
7. **Grounding:** each of FORM, Expert evolution, and non-trivial component claims carries `grounding` + `source`; `mnemonic-device` / `speculative` render visible marker (Encoding PRD parity). *Check: linter rejects untagged Expert prose.*
8. **Outlier firewall:** build asserts no Outlier PDF prose is copied verbatim into shipped JSON (similarity check on reference run); PDF paths never appear in public repo. *Check: grep + one fixture.*

**Integration:**

9. Reader hover "character" link on single-char tokens → `#/dict/<char>`. Encoding page character story imports `resolveDictAppEntry(char)` summary. *Check: 熱鬧 → tap 鬧 → dict page.*
10. Word store: optional `status` / `flag` on character entries for "studying this component series." *Check: optional, does not block v1.*

## 4. Users & use cases

**Primary: Wedge** — owns Outlier PDFs; learns zh-Hant via immersion + SRS; wants Outlier-grade understanding without leaving Tsumugu.

1. **Deep lookup while reading.** Unknown character in a passage → open Dictionary → read FORM + COMPONENTS + hear Serena → return to reader with predictive guess for sound/meaning.
2. **Sound-series study.** Browse `#/dict/series/戠` → see 識, 職, 織, … → learn the pattern once.
3. **Expert pass.** Read evolution + ancient forms for 造; compare to 告; understand false friends.
4. **Pipeline study (Lesson 12).** Study mode: show FORM+COMPONENTS first; reveal MEANINGS on tap; reveal Expert last — spaced within one session.
5. **Batch build.** `pnpm gen dict-app --chars 造,識,鬧 --tier expert` → agent uses PDF index + pedagogy prompts → verify → `gen dict-voice` for Serena → promote to vault.
6. **Correct.** Edit `custom` override in vault; precedence wins per field.

## 5. Core design

### 5.1 Entry schema (`tsumugu/dict-entry@1`)

```ts
export type ComponentRole = "form" | "meaning" | "sound" | "empty";

export interface DictComponent {
  order: number;              // 1, 2, … as in expert-details.jpg
  glyph: string;
  role: ComponentRole;
  reading?: string;
  /** In 造, 辶 is a form component depicting a foot walking… */
  function: string;
  meaningLabel?: string;
  soundHint?: string;
  isCorrupted?: boolean;
  corruptionNote?: string;
  grounding: "sourced" | "mnemonic-device" | "speculative";
  source?: string;
  audio?: string;             // Serena ref: sections/comp-01.mp3
}

export interface MeaningNode {
  id: string;
  sense: string;              // English or 繁中 per display setting
  senseZh?: string;
  kind: "original" | "extended" | "modern" | "sound-loan" | "derived";
  pronunciation?: string;     // e.g. ancient cào vs modern zào
  children?: MeaningNode[];
}

export interface AncientForm {
  label: string;              // "oracle", "bronze", "Warring States", …
  glyph: string;              // Unicode or private image ref in vault
  note?: string;
}

export interface SystemLink {
  char: string;
  relation: "same-sound-component" | "same-meaning-component" | "same-form-component" | "compare" | "radical-category";
  note?: string;
}

export interface DictEntryExpert {
  /** Long-form evolution (Expert column in expert-details.jpg). */
  evolution: string;
  ancientForms?: AncientForm[];
  comparisons?: { char: string; note: string }[];
  phonology?: { oldChinese?: string; middleChinese?: string; mandarin: string };
  citations?: string[];
  audio?: string;
  grounding: "sourced" | "mnemonic-device" | "speculative";
  source?: string;
}

export interface DictAppEntry {
  schema: "tsumugu/dict-entry@1";
  char: string;               // NFC single character (primary key)
  reading: string;            // primary Mandarin (Taiwan)
  zhuyin: string;
  tone: number;
  /** Unit I — three attributes one-liner */
  attributesSummary?: string;
  /** FORM section */
  formExplanation: string;
  formAudio?: string;
  /** COMPONENTS section */
  components: DictComponent[];
  /** MEANINGS / meaning tree */
  meaningTree: MeaningNode[];
  meaningsAudio?: string;
  /** STROKE tab */
  strokeCount: number;
  strokeOrder: string[];      // glyph names or indices
  /** Expert block (required for expert-tier generation) */
  expert: DictEntryExpert;
  /** SYSTEM — CHARS tab */
  systemLinks: SystemLink[];
  /** WORDS tab — headwords containing this char (from pack index / word store) */
  words?: { term: string; reading?: string; gloss?: string }[];
  /** SENTS tab — 3–6 example sentences (Traditional + EN); band-gated optional */
  sentences?: { text: string; translation: string; audio?: string }[];
  pedagogy?: { levelOfUnderstanding: number; pipeliningHint?: string };
  generatedAt: string;
  generator: "agent-v1";
}
```

### 5.2 Generation pipeline (staged agent)

Aligned with Masterclass order — **do not skip stages**.

| Stage | Prompt | Input | Output |
|-------|--------|-------|--------|
| **G0 — Index** | — | PDF `index.json` + char | PDF page pointers for agent |
| **G1 — Deep structure** | `dict-app-structure.md` | char, PDF excerpt, glossary | `components[]`, `formExplanation`, roles typed |
| **G2 — Meanings** | `dict-app-meanings.md` | G1 + PDF 09 rules | `meaningTree` |
| **G3 — Expert** | `dict-app-expert.md` | G1–G2 + PDF excerpt | `expert.*`, ancient forms, comparisons |
| **G4 — System** | `dict-app-system.md` | G1 sound component | `systemLinks`, sound series |
| **G5 — Satellite** | `dict-app-satellite.md` | CC-CEDICT + word store | `words[]`, `sentences[]` |
| **G6 — Verify** | `gen verify dict` | full entry | grounding, OpenCC, role completeness, Expert presence |
| **G7 — Voice** | `gen dict-voice` | entry text fields | Serena MP3s + manifest `dict-audio@1` |

**Reference firewall (G1–G4):** prompts may quote **facts** (component role, OC reading) from PDFs; they may not paste **prose paragraphs**. Agent rewrites in Tsumugu register (Traditional Taiwan, 教育部國語小字典-ish clarity for Chinese UI strings).

**STROKE tab:** stroke order build-up (data from entry or shared stroke pack).  
**CHARS tab:** `systemLinks` grouped by relation (sound series first).  
**WORDS / SENTS:** satellite data; link to reader prepared content when available.

### 5.5 Study modes (Pipelining — PDF 12)

## 6. Architecture

| Layer | Location | Notes |
|-------|----------|-------|

## 9. Risks & mitigations

| Risk | Mitigation |
|------|------------|
| AI invents false etymology | Grounding lint; Expert requires citations or `speculative` marker; human review queue on verify fail |
| Outlier prose leakage | Similarity check vs reference; firewall in prompts; private data only |
| OC/MC errors | Label phonology as generated; optional `confidence`; don't assert in UI without marker |
| Expert scope creep | Schema requires Expert block but allows shorter `evolution` for simple pictographs |
| Audio cost/time | Incremental `gen dict-voice`; section-level not sentence-by-sentence for Expert prose splits |
| Duplication with Encoding | Encoding imports projection; Dictionary App is canonical for character depth |
